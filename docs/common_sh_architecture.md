## 类比：生活中的例子

### 🏠 建房子

**没有共享库** = 每座房子都要自己制造砖块、水泥、钢筋
- 效率低
- 质量不一致
- 成本高

**有共享库** = 从建材市场购买标准化材料
- 效率高
- 质量有保证
- 成本低

### 🍳 做菜

**没有共享库** = 每次做菜都要从头准备调料、清洗工具
- 浪费时间
- 容易出错

**有共享库** = 使用准备好的调料盒、清洁的工具
- 节省时间
- 标准化流程

### 📚 写作

**没有共享库** = 每本书都重新定义标点符号、语法规则
- 读者困惑
- 交流困难

**有共享库** = 使用统一的语言标准
- 清晰明确
- 易于理解

## 关键要点

> 💡 **common.sh 是"智能重用"，不是"额外负担"**

1. **写一次，到处使用** - 85 行代码，服务所有脚本
2. **修改一次，到处更新** - 一个 bug 修复，所有脚本受益
3. **测试一次，全部验证** - 集中测试，覆盖所有用例
4. **学习一次，终身受用** - 理解一次，应用于所有新脚本

这就是为什么我们需要 `tools/common.sh`！
