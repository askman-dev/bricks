# Background
用户反馈聊天界面存在频道（上级实体）与分区（下级实体）ID 混用问题，并且两个菜单缺少点击空白关闭行为，输入框菜单弹层定位间距过大，同时对话标题错误居中。

# Goals
1. 修复右上角分区列表中错误使用频道 ID 的问题，确保分区 ID 与频道 ID 不再混淆。
2. 修复右上角分区菜单：点击空白区域时自动关闭。
3. 修复左下角输入框菜单：点击空白区域时自动关闭。
4. 修复左下角输入框菜单定位：紧贴触发按钮上方显示。
5. 修复对话标题“Bricks”布局：从居中改为左对齐。
6. 变更后同步检查并按需更新代码地图。

# Implementation Plan (phased)
1. 定位频道/分区数据模型与 UI 渲染文件，确认 ID 来源与映射逻辑。
2. 修复分区菜单条目绑定逻辑，确保使用分区实体自身 ID。
3. 为右上角分区菜单与左下角输入框菜单添加一致的 outside click 关闭机制。
4. 调整输入框菜单弹出定位规则（anchor/origin/offset/z-index 等）以贴近触发按钮上沿。
5. 调整标题容器布局样式为左对齐，并检查打开侧边栏后的视觉一致性。
6. 运行相关检查/测试。
7. 根据变更影响检查并更新 `docs/code_maps/feature_map.yaml` 与 `docs/code_maps/logic_map.yaml`（如需要）。

# Acceptance Criteria
- 右上角分区菜单中每个分区条目操作都基于分区 ID，不再与侧边栏频道 ID 相同。
- 右上角分区菜单打开后，点击菜单外任意空白区域会关闭。
- 左下角输入框菜单打开后，点击菜单外任意空白区域会关闭。
- 左下角输入框菜单弹层紧贴触发按钮上方，不再存在明显大间距。
- 标题“Bricks”显示为左对齐，打开侧边栏时与侧边栏文本对齐风格一致。
- 若代码地图受影响，已完成同步更新并通过基础 YAML 校验命令。
