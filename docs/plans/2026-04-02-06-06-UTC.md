# OAuth state validation fix for preview-domain login callback

## Background
Users starting the web app from a Vercel preview URL are being redirected to the production OAuth callback host, and the callback can fail with `{"error":"Invalid or missing OAuth state parameter"}` when the nonce cookie is not available on that host.

## Goals
- Keep CSRF protections for standard same-host OAuth flows.
- Allow callback validation to succeed for cross-domain preview -> production callback flows.
- Preserve backwards compatibility for in-flight/legacy OAuth state values.

## Implementation Plan (phased)
1. Add signed OAuth state support (HMAC) so callback validation can be stateless when the cookie is missing.
2. Keep cookie-based nonce validation as the primary check when the cookie is present.
3. Update callback validation logic to accept either:
   - cookie nonce match, or
   - valid state signature when cookie is absent.
4. Add/extend unit tests for signed state validation logic.

## Acceptance Criteria
- Starting OAuth from a preview domain and returning to production callback no longer fails solely due to missing `oauth_state` cookie.
- Same-host OAuth still validates nonce against the cookie.
- Existing tests pass and new tests cover signed-state verification behavior.
