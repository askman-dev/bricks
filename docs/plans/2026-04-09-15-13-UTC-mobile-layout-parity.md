# Background
用户提供了主干分支的移动端截图，当前 React 前端布局与截图在结构、间距、颜色、文案和导航交互上存在明显偏差（例如当前使用深色桌面侧栏）。本任务需要按移动端优先重建关键页面布局，目标是与截图视觉一致，并逐项核对布局。

# Goals
1. 将 Chat、Settings 页面改为移动端优先布局，匹配截图的顶部栏、列表项、输入框、抽屉/菜单样式与颜色。
2. 保留现有路由与核心交互能力（发送消息、进入模型设置、退出登录）。
3. 更新测试以覆盖新布局中的关键可见元素与发送流程。
4. 评估并同步代码地图（feature_map.yaml / logic_map.yaml）是否需要更新。

# Implementation Plan (phased)
## Phase 1: 结构改造
- 重构 `App.tsx`，移除桌面侧栏，改为仅内容区路由渲染。
- 重构 `ChatPage.tsx` 为移动端容器，增加顶部导航栏、聊天列表、底部输入区域及菜单弹层。
- 重构 `SettingsPage.tsx` 为截图风格的设置列表页（标题、返回箭头、三项列表）。

## Phase 2: 样式对齐
- 在 `styles.css` 中新增/替换移动端主题色、间距、字体和圆角，确保与截图接近。
- 细化菜单、下拉框、条目二级文案、图标占位与分隔线样式。

## Phase 3: 质量验证
- 调整前端单测以匹配新 UI 文案和结构。
- 运行 `npm test`（web_chat_app）验证回归。
- 检查代码地图是否需更新；如无需更新，在最终说明给出理由。

# Acceptance Criteria
1. `/chat` 页面在移动宽度下呈现浅灰背景、顶部栏“Bricks”、消息卡片和底部大输入框，视觉结构与截图一致。
2. `/settings` 页面显示“Settings”标题，包含 Model Settings / Manage Agents / Sign Out 三个条目及副标题文案。
3. 发送消息流程仍可将用户消息与助手回复追加到消息区。
4. `apps/web_chat_app` 单元测试通过。
